# カスタムクイズ問題セットの作り方

QUIZアプリでは、JSON形式で書いた問題セットをURLから取り込んで、通常のクイズカテゴリと同じように遊べます。コードは一切実行されず、テキスト・画像URLなどの「データ」だけを読み込む仕組みなので安全です。

## 1. JSONをホスティングしてURLを発行する

アプリはURLに直接アクセスしてJSONを取得するので、**生のJSON本文が返ってくるURL**が必要です（HTMLページのURLではなく）。

### GitHub Gist を使う（おすすめ）

1. https://gist.github.com/ を開く（GitHubアカウントでログイン）
2. ファイル名を `quiz.json` などにして、下記の形式でJSONを貼り付ける
3. 公開する場合は「Create public gist」を選択（QUIZから取り込むにはpublicである必要があります）
4. 作成後、ファイル右上の「Raw」ボタンを押すと表示されるURL
   （`https://gist.githubusercontent.com/ユーザー名/xxxxx/raw/quiz.json` のような形）をコピー

Gistを編集した場合は、Rawボタンから改めてURLを取得し直してください（コミットごとにURLが変わることがあります）。

### その他の選択肢

- [JSONBin.io](https://jsonbin.io/) — JSON専用のホスティングAPI
- [npoint.io](https://www.npoint.io/) — JSON専用のホスティング、簡易エディタ付き
- 自分のGitHubリポジトリに `.json` ファイルを置いて `raw.githubusercontent.com` のURLを使う

## 2. JSONの形式

```json
{
  "label": "問題セット名",
  "description": "説明文",
  "questions": [
    {
      "mode": "choice",
      "prompt": "この問題文が表示されます",
      "answer": "正解の選択肢",
      "image": "https://example.com/image.png"
    },
    {
      "mode": "text",
      "prompt": "名字を入力してください",
      "answers": ["正解1", "正解1の別表記"]
    },
    {
      "mode": "date",
      "prompt": "誕生日は何月何日？",
      "month": 1,
      "day": 1
    }
  ]
}
```

### トップレベルのフィールド

| フィールド | 型 | 必須 | 説明 |
| --- | --- | --- | --- |
| `label` | string | ○ | カテゴリ一覧に表示される名前（1〜60文字） |
| `description` | string | - | カテゴリ一覧に表示される説明文（200文字まで、省略時は空欄） |
| `questions` | array | ○ | 問題の配列（1〜200件） |

### 問題（`questions`の各要素）に共通のフィールド

| フィールド | 型 | 必須 | 説明 |
| --- | --- | --- | --- |
| `mode` | `"choice"` \| `"text"` \| `"date"` | ○ | 出題形式 |
| `prompt` | string | ○ | 問題文 |
| `image` | string | - | 画像の**URL**（`http://`または`https://`から始まる必要があります） |

### `mode: "choice"`（4択問題）

| フィールド | 型 | 必須 | 説明 |
| --- | --- | --- | --- |
| `answer` | string | ○ | 正解の文字列。他の問題の`answer`からランダムに3つ選んでダミー選択肢にします |

### `mode: "text"`（自由入力問題）

| フィールド | 型 | 必須 | 説明 |
| --- | --- | --- | --- |
| `answers` | string[] | ○ | 正解として許容する文字列の配列（表記ゆれ対応。1つ以上） |

### `mode: "date"`（月日選択問題）

| フィールド | 型 | 必須 | 説明 |
| --- | --- | --- | --- |
| `month` | number | ○ | 正解の月（1〜12） |
| `day` | number | ○ | 正解の日（1〜31） |

## 3. アプリへの取り込み方

1. QUIZアプリを開く
2. カテゴリ一覧の一番下にある「＋ カスタム問題セットを追加」をタップ
3. 発行したJSONの生URLを入力して「追加」
4. 問題なく読み込めれば、カテゴリ一覧にバッジ付きで追加されます

- 同じURLをもう一度追加すると、内容が上書きされます（統計もそのまま引き継がれます）
- 各カテゴリカードの右上の「✕」から個別に削除できます
- カスタム問題セットは端末ごとに保存されます（他の端末とは共有されません）
- 最大10セットまで追加できます

## 4. 制限・エラーになる主なケース

- URLが `http://` または `https://` で始まっていない
- JSONとして解析できない、またはサイズが大きすぎる（約500KB超）
- `label`が空文字または60文字を超えている
- `questions`が空配列、または200件を超えている
- 各問題の`mode`・必須フィールドが不足/不正（`choice`なら`answer`、`text`なら`answers`、`date`なら`month`/`day`）

これらに該当する場合、追加時にエラー内容がダイアログで表示されます。
